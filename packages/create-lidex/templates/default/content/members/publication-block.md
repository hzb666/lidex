Not every content type needs detail Markdown. Sometimes a strong list layout, a small fixed field set, and a link to an external resource are enough. The publication block exists to show that Lidex can support those cases without forcing everything through one detail-page model.

That keeps the configuration honest: route-bearing content types declare routes, while lightweight reference content types remain simple cards with their own template.
